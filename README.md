Project OverviewCrop diseases pose a massive threat to global food security, often leading to devastating yield losses if not identified and treated early. FloraGuard is a comprehensive, end-to-end Machine Learning ecosystem designed to automatically identify, classify, and visualize plant leaf diseases from standard digital images.By leveraging state-of-the-art Convolutional Neural Networks (CNNs) and Transfer Learning, this project provides a robust, highly accurate diagnostic tool accessible via both a RESTful API and an interactive web interface. It is built to assist agronomists, farmers, and researchers in making data-driven agricultural decisions.📑 Table of ContentsKey FeaturesSystem ArchitectureDataset DetailsTechnology StackInstallation & SetupUsage GuideAPI DocumentationModel PerformanceFuture RoadmapContributing✨ Key FeaturesExtensive Disease Library: Accurately classifies over 38 distinct classes of plant leaves (including healthy leaves and specific diseases like Apple Scab, Tomato Early Blight, Corn Northern Leaf Blight, etc.).High-Precision Inference: Built on a deeply fine-tuned ResNet50 architecture, optimized to handle complex background noise and varying lighting conditions.Visual Explainability (Grad-CAM): Demystifies the AI's decision by generating heatmaps over the original image, highlighting the exact diseased lesions the model focused on.High-Performance Backend: Utilizes FastAPI for asynchronous, rapid-fire inference capable of handling multiple requests simultaneously.Interactive Dashboard: A Streamlit-based web app that allows drag-and-drop image uploading, real-time prediction viewing, and confidence score charting.Containerization: Fully Dockerized for seamless deployment across any cloud provider (AWS, GCP, Azure) or local server.🏗 System ArchitectureThe system is divided into three primary pipelines:Data Pipeline:Raw images are ingested, resized to 224x224, normalized (pixel values scaled between 0 and 1), and augmented (rotation, zooming, horizontal flipping) to prevent model overfitting.Inference Engine:Images pass through the globally cached ResNet50 model. The final dense layer utilizes a Softmax activation function to output a probability distribution across all 38 classes.Application Layer:The FastAPI backend exposes the inference engine via a /predict endpoint. The Streamlit frontend queries this endpoint and renders the results, including the generated Grad-CAM heatmap, to the end user.📊 Dataset DetailsThe model was trained using the renowned PlantVillage Dataset, supplemented with custom field images to improve real-world robustness.Total Images: 54,305Train/Validation/Test Split: 80% / 10% / 10%Species Covered: 14 (Apple, Blueberry, Cherry, Corn, Grape, Orange, Peach, Bell Pepper, Potato, Raspberry, Soybean, Squash, Strawberry, Tomato)Data Augmentation: Applied random brightness, contrast adjustments, and shear transformations during training to simulate different field capture conditions.🛠 Technology StackMachine Learning & Computer VisionTensorFlow / Keras: Deep learning framework for model building and training.OpenCV & PIL: Image processing, matrix transformations, and bounding box creation.tf-keras-vis: For Grad-CAM implementation.Backend & APIFastAPI: High-performance web framework for building the REST API.Uvicorn: ASGI web server implementation.Pydantic: Data validation and settings management.Frontend & DeploymentStreamlit: Rapid UI development for the user dashboard.Docker & Docker Compose: Container orchestration.Gunicorn: Production-grade WSGI HTTP server.🚀 Installation & SetupPrerequisitesPython 3.8, 3.9, or 3.10Git(Optional) Docker and Docker ComposeOption 1: Local Development Setup1.
